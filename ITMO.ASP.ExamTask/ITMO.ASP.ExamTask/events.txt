Вместе дешевле

=== Задача ===
Разработать веб-приложение. Требования по техническому стеку: 
 - web-forms / mvc / web api (на выбор);
 - БД - mssql  или др. (при обосновании технической целесообразности);
Оформление: исходный код опубликован в git-репозитории. Файл справки (.md) содержит всю необходимую информацию для сборки, запуска, отладки приложения (настройка окружения, сети, БД, зависимости)

=== Критерии оценки (по приоритету) ===
 - работоспособность (приложение собирается, запускается без ошибок);
 - качество решения инженерной задачи (приложение выполняет функции описанные в постановке задачи);
 - стилистика исходного кода;
 - качество демонстрации решеной задачи (демонстрация работы, короткий рассказ о технических делатлях реализации, какие были проблемы, как были решены);
 - инициативность, дополнительная самостоятельная работа;
 - оформление репозитория и документирование.

=== Описание ===
 
Приложение позволяет людям организовываться в группы фиксированного размера (напр. для получения скидок за групповое посещение). 2 типа пользователей - организатор и участник. Организатор размещает запрос на формирование группы, участники просматривают список открытых групп и подтверждают своё участие.
 
Организатор:
 - создаёт объявление о формировании группы. Поля: кол-во человек, описание, фотограии, дата начала события, дата окончания, теги;
 - просматривает список своих объявлений (в т.ч. завершенные и отмененные)
 - подтверждает / отклоняет откликнувшихся участников;
 - может отменить событие (максимум за сутки до начала);
 - не может создавать пересекающиеся события;
 - может самостоятельно регистрироваться в системе;
 - после завершения события организатор может оставить отметку о каждом участнике (явился / не явился);
 - Видит свой рейтинг;
Участник:
 - может просматривать список открытых объявлений (начало еще не наступило и есть вакансии);
 - может искать события по тегу;
 - может просматривать рейтинг организатора события и сортировать / фильтровать список поиска по рейтингу;
 - После завершения события каждый участник может оставить отметку о событии (состоялось успешно / не состоялось)
 - Не может участвовать более чем в одном пересекающемся событиии, но может откликаться на пересекающиеся события;
 - Для упрощения реализации, в случае конкуретных заявок - участие в событии определяется организатором (первый подтвердивший участие организатор "получает" участника,остальные заявки отменяются с пометкой);
 - Может отменять заявку на участие если она не подтверждена;
 - Может просматривать историю своих заявок (в т.ч. отмененные им самим и отмененные автоматически);
 - Видит свой рейтинг ;
Общие требования:
  - Пользовательский интерфейс содержит компонент обратного отсчета до начала события;
  - Предусмотрен постраничный вывод для данных;
  - в системе может быть неограниченное количество организаторов и участников;
  - Каждый организатор может войти в систему как участник и наоборот с помощью одной и той же учетной записи, но приложение доступно только в контексте роли выбранной при авторизации;
  - Предложена формула расчета рейтинга организатора и участника на основе данных об участии в событиях;
  - Приложение должно иметь тестовый набор данных для демонстрации всех требований и граничных сценариев;
